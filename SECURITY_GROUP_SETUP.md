# 🔒 AWS Security Group 설정 가이드

## 현재 상황
- 인스턴스 상태: ✅ 실행 중
- 보안 그룹: `default`
- 문제: 포트 22, 80, 443이 닫혀있음

## 해결 방법

### 1. AWS Console에서 Security Group 설정

1. **EC2 Console 접속**
   - https://console.aws.amazon.com/ec2/
   - 왼쪽 메뉴에서 "보안 그룹" 클릭

2. **default 보안 그룹 선택**
   - `default` 보안 그룹 찾기
   - 또는 인스턴스에서 "보안" 탭 → 보안 그룹 이름 클릭

3. **인바운드 규칙 편집**
   - "인바운드 규칙" 탭 클릭
   - "인바운드 규칙 편집" 버튼 클릭
   - "규칙 추가" 클릭

4. **필요한 포트 추가**

   | 유형 | 프로토콜 | 포트 범위 | 소스 | 설명 |
   |------|---------|----------|------|------|
   | SSH | TCP | 22 | 내 IP 또는 0.0.0.0/0 | SSH 접속 |
   | HTTP | TCP | 80 | 0.0.0.0/0 | 웹 서버 |
   | HTTPS | TCP | 443 | 0.0.0.0/0 | HTTPS |
   | Custom TCP | TCP | 3001 | 0.0.0.0/0 | 백엔드 API |

5. **규칙 저장**
   - "규칙 저장" 클릭

### 2. 빠른 설정 (전체 공개 - 개발용)

개발 단계에서는 모든 IP에서 접속 허용:
- SSH (22): 0.0.0.0/0
- HTTP (80): 0.0.0.0/0
- HTTPS (443): 0.0.0.0/0
- Custom (3001): 0.0.0.0/0

### 3. 보안 그룹 설정 확인

설정 후 몇 분 기다린 다음:
```bash
./server-check.sh
```

## ⚠️ 중요 사항

- **프로덕션 환경**에서는 SSH 포트는 본인 IP만 허용하는 것이 좋습니다
- 현재는 **개발/테스트 단계**이므로 모든 IP 허용 가능
- 보안 그룹 변경 후 **1-2분** 정도 기다려야 적용됩니다
